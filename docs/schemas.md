# TasteKit Artifact Schemas

All data in TasteKit is defined by a set of versioned, machine-readable schemas. These schemas are the canonical source of truth for agent behavior and ensure that artifacts are portable and interoperable across different runtimes. All artifacts are stored in either JSON or YAML format.

## Core Artifacts

The following core artifacts are generated by the `tastekit compile` command and stored in the `.tastekit/artifacts/` directory.

### `constitution.v1.json`

This is the most important artifact, representing the user's global taste profile.

| Field | Type | Description |
| :--- | :--- | :--- |
| `schema_version` | String | The version of the schema, e.g., `"constitution.v1"`. |
| `principles` | Array | An ordered list of core principles that guide the agent's behavior. |
| `tone` | Object | Defines the agent's communication style, including voice keywords and forbidden phrases. |
| `tradeoffs` | Object | Specifies user preferences for tradeoffs like accuracy vs. speed and autonomy level. |
| `evidence_policy` | Object | Rules for when and how to cite sources or express uncertainty. |
| `taboos` | Object | Defines actions the agent must never take and situations that require human escalation. |

### `guardrails.v1.yaml`

This artifact defines the security and permission model for tool usage.

| Field | Type | Description |
| :--- | :--- | :--- |
| `schema_version` | String | The version of the schema, e.g., `"guardrails.v1"`. |
| `permissions` | Array | A list of rules defining which tools can access which resources. |
| `approvals` | Array | A set of rules that trigger an approval workflow before an action is executed. |
| `rate_limits` | Array | Defines usage limits for specific tools to prevent abuse or excessive cost. |

### `memory.v1.yaml`

This artifact defines the policy for how the agent's memory should be managed by the runtime.

| Field | Type | Description |
| :--- | :--- | :--- |
| `schema_version` | String | The version of the schema, e.g., `"memory.v1"`. |
| `write_policy` | Object | Rules for determining what is important enough to remember (salience). |
| `retention_policy` | Object | Rules for how long to retain memories and when to prune them. |
| `consolidation_schedule` | String | A cron expression defining when to run memory consolidation tasks. |

## Supporting Artifacts

These artifacts are also critical to the TasteKit ecosystem.

### `bindings.v1.yaml`

This file, located in `.tastekit/`, stores the configuration for all connected MCP (Model Context Protocol) servers and the specific tools, resources, and prompts that have been bound to the workspace.

### `trust.v1.yaml`

Located in `.tastekit/`, this file defines the trust policy for the workspace. It contains the cryptographic fingerprints of pinned MCP servers and skill sources, ensuring that the agent only connects to and uses trusted resources.

### `playbook.v1.yaml`

Playbooks are runtime-agnostic plans that define a sequence of steps for an agent to follow to accomplish a specific task. They are stored in `.tastekit/artifacts/playbooks/` and can be triggered by events, schedules, or manual invocation.

### `evalpack.v1.yaml`

Evaluation packs, stored in `.tastekit/artifacts/evalpacks/`, contain a set of scenarios and judging criteria used to evaluate an agent's performance and alignment with the user's taste.

### `trace_event.v1.jsonl`

Trace files are stored in `.tastekit/traces/` as JSONL files (one JSON object per line). Each line is a `trace_event.v1` object that records a single event in the agent's lifecycle, such as a tool call, an approval request, or an error.
