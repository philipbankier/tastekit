What is TasteKit?

TasteKit is a CLI tool + library that interviews you about your preferences, compiles them into structured artifacts, and exports them to any AI agent runtime. One interview. Every runtime. Your taste, everywhere.

Core

Interview-Driven

An LLM-powered interviewer adapts its questions based on your answers. It tracks 9–28 dimensions of your preferences across 3 depth tiers (5min / 15min / 30min).

Core

Artifact-First

Everything compiles to human-readable, git-friendly JSON/YAML files. No database. No runtime dependency. Version control your agent's personality like code.

Unique

Drift Detection

The only OSS tool that detects personality-level drift. 7 signal types monitor your agent's behavior against its constitution and propose corrections.

Core

Portable Profiles

Skills follow the Agent Skills open standard (agentskills.io) — compatible with 32+ runtimes: Claude Code, OpenClaw, OpenAI Codex, Gemini CLI, Cursor, VS Code Copilot, GitHub, Junie, Letta, ActRun Autopilots, and more.

Core

MCP-First Tools

Wraps the official MCP SDK. Trust pinning with cryptographic fingerprints. Auditing catches silent tool changes. 17,665+ MCP servers available.

New

Agent Skills Spec

Skills follow the Agent Skills open standard (agentskills.io). Progressive disclosure (3 levels), YAML frontmatter, references/ directory. Portable across all 32+ compatible runtimes.

Drift Detection — TasteKit's Unique Differentiator

No other open-source tool detects personality-level drift. TasteKit monitors 7 signal types, proposes corrections with risk ratings, and supports accept/reject workflows.

🔄

Repeated Edit

User repeatedly rejects agent actions (threshold: ≥3). Signals misalignment with preferences.

⚠️

Principle Violation

Error accumulation suggests principles are being broken (threshold: ≥3 errors).

✏️

User Correction

User edits agent output within 5 minutes. Frequent corrections mean the agent isn't matching taste.

🔧

Tool Change

User switches tools for the same skill. Suggests tool bindings need updating.

📅

Staleness

Constitution age >30 days. Auto-triggered — preferences evolve, profiles should too.

🕳️

Coverage Gap

Tools/skills in use but not in manifest. The agent learned new tricks that aren't tracked.

🚫

Assertion Mismatch

"Never do X" principle exists but traces show X happening. Direct constitution violation.

Constitution Schema

The constitution is the core artifact — it captures who you are as a builder. Every other artifact references it.

constitution.v1.json ├── schema_version: "constitution.v1" ├── generated_at: ISO8601 ├── generator_version: string ├── user_scope: "single_user" ├── principles: ConstitutionPrinciple[] │ ├── id: string // e.g., "p-readability" │ ├── priority: number // 1 = highest │ ├── statement: string required │ ├── rationale: string? │ ├── applies_to: string[] // ["*"] or ["code", "docs"] │ ├── examples_good: string[]? │ └── examples_bad: string[]? ├── tone: ConstitutionTone │ ├── voice_keywords: string[] // ["direct", "conversational"] │ ├── forbidden_phrases: string[] // ["synergy", "leverage"] │ └── formatting_rules: string[] ├── tradeoffs: ConstitutionTradeoffs │ ├── accuracy_vs_speed: 0.0–1.0 // 0=speed, 1=accuracy │ ├── cost_sensitivity: 0.0–1.0 │ └── autonomy_level: 0.0–1.0 // → Claude Code defaultMode ├── evidence_policy │ ├── require_citations_for: string[] │ └── uncertainty_language_rules: string[] ├── taboos │ ├── never_do: string[] │ └── must_escalate: string[] └── trace_map: Record<string, any> // source traceability

CLI Commands

12 commands covering the full lifecycle: init → interview → compile → export → monitor → maintain.

tastekit init

Create workspace. Pick domain, depth, LLM provider. Creates .tastekit/ with three-space layout.

tastekit onboard

LLM-driven interview. Adaptive questions, coverage tracking, /save to resume later. Voice mode optional.

tastekit compile

Generate 5 artifact types from session. Supports --resume for interrupted compilations.

tastekit export

Export to claude-code, openclaw, manus, autopilots, agents-md, or agent-file formats.

tastekit skills

list, lint, graph, pack, report, inspect, history, rollback — full skill lifecycle management.

tastekit drift

detect, review, accept, reject — monitor agent behavior against constitution.

tastekit mcp

add, list, inspect, bind — manage MCP tool servers with trust pinning.

tastekit trust

init, pin-mcp, pin-skill-source, audit — cryptographic trust management.

tastekit eval

run, replay — execute evaluation packs against compiled artifacts.

tastekit import

Import from SOUL.md, IDENTITY.md, or Letta Agent File (.af) format.

tastekit completion

Generate shell completion scripts for bash, zsh, and fish.

tastekit simulate

Run skill simulations with dry-run and no-side-effects modes.

System Architecture

5 packages, 9 artifact types, 4 adapters. The core compiles artifacts. Adapters export them. Drift detection closes the loop.
